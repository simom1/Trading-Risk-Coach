Kaggle Capstone Project

Trading Risk Coach

一款专为个人交易者研发的多智能体风控诊断与行为去偏副驾驶系统，不提供投资建议，只做交易行为复盘与风险控制辅助。

# 解决的核心痛点 (What Problems it Solves)

📈

处置效应 (Disposition Effect)

散户交易者在面对盈利时倾向于急切“落袋为安”（赚小钱就跑），而在面对亏损时却长期死扛（扛单）甚至逆势加仓，导致总期望收益为负。

🛡️

量化风险意识缺失

普通投资者缺乏对下行风险敞口的科学量化手段，无法计算单笔最大风险暴露（VaR/CVaR）以及评估平台和资产集中度风险。

🚦

情感决策与AI幻觉

交易者在连续亏损时易陷入“赌徒谬误”进行非理性补仓。同时，通用AI大模型给出的风控建议也易产生幻觉，误导用户做出高风险操作。

# 系统工作流程 (Agentic Pipeline)

点击下方每一个运行步骤，查看具体的底层执行逻辑与对应的代码文件：

1
数据加载与指标计算 Analysis Agent

系统启动后，调用 MCP 数据服务，拉取交易流水并科学计算核心量化指标。

对应文件：trading_risk_coach/agents/analysis_agent.py
逻辑：通过 Stdio 连接标准 MCP 数据包，执行 Python 进行胜率、盈亏比（平均盈利 vs 平均亏损绝对值）、单笔亏损极值的数学计算，完成Observe（观察）阶段。
2
规则注入与风险诊断 Advisor Agent

对比行为金融学 Skill 规范，判定是否存在偏差，并针对高危活跃持仓触发风控干预。

对应文件：trading_risk_coach/agents/advisor_agent.py
逻辑：读取 SKILL.md 规范，若判定“处置效应系数 = 平均亏损绝对值 / 平均盈利 >= 1.5”则触发赢小亏大警报。若发现活跃持仓中存在未挂止损高危单，将直接调用风控工具（设置硬止损/紧急平仓），完成Think & Act（思考与行动）阶段。
3
零信任硬拦截安全过滤 Guardrail Callback

在风控决策输出给用户之前，由确定性 Python 熔断护栏过滤拦截潜在的危险投资建议。

对应文件：trading_risk_coach/guardrails/safety_rules.py
逻辑：大模型生成的文本经过注册的 after_model_callback 拦截。如果文本中包含任何“加仓回本”、“摊平成本”等赌徒谬误词汇，系统会强行抹除并替换为标准的红色安全警报文本，实现物理级别零信任拦截。
4
审计排版与风控意见书呈献 Critic Agent

审计生成的报告以确保指标准确度，进行三态风控评级分类，并生成工整的报告。

对应文件：trading_risk_coach/agents/critic_agent.py
逻辑：对前序报告执行 QA 审查，确切核对每一项数字指标是否具备数据依据，并根据安全状态将其分类为：🟢 【绿灯 - 安全】、🟡 【黄灯 - 警报观察】、🔴 【红灯 - 熔断执行】，产出终审报告。

# Demo 场景与未来改进 (Demo & Roadmap)

示例输入：

Please review my recent XAUUSD trades and tell me whether I am showing win-small-lose-big behavior.

系统预期执行链路：

MCP 读取交易流水 -> Analysis Agent 计算胜率/平均盈亏/处置效应系数 -> Advisor Agent 对照 SKILL.md 触发风控规则 -> 必要时调用 execute_risk_mitigation 设置硬止损 -> Guardrail 拦截危险恢复建议 -> Critic Agent 输出三态风控报告。

局限性与未来方向：

当前经纪商执行层为模拟工具，数据层使用匿名化 MT5 交易导出和 XAUUSD M1 行情 CSV。后续可替换为真实券商/交易所 API、数据库型 MCP Server、组合级敞口汇总、更多回撤指标，以及面向风控人员的操作仪表盘。
