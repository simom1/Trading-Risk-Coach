Trading Risk Coach

A multi-agent risk diagnostics and behavioral de-biasing copilot built for retail traders. It does not provide investment advice, only trading review and risk control assistance.

Technical Docs Replay Sandtable KPI Dashboard

# What Problems it Solves

📈

Disposition Effect

Retail traders tend to take profits quickly ("win small") to avoid giving back gains, but hold onto losing positions ("扛单") or average down against the trend, resulting in negative expected returns.

🛡️

Lack of Quantitative Risk Awareness

Average investors lack scientific tools to quantify downside exposure, calculate VaR limits, or evaluate symbol and platform concentration risks.

🚦

Emotional Decisions & AI Hallucinations

Traders often fall into the "Gambler's Fallacy" and add to losing trades. Furthermore, general LLMs often hallucinate and recommend high-risk averaging-down behaviors.

# System Workflow (Agentic Pipeline)

Click each execution step below to view the underlying logic and corresponding code files:

1
Data Loading & Metric Calculation Analysis Agent

Once started, the system calls MCP data tools to retrieve trade logs and compute core quantitative performance metrics.

File: trading_risk_coach/agents/analysis_agent.py
Logic: Connects via Stdio to stdio MCP tools, executes Python pandas to calculate win rate, reward-to-risk ratio, and maximum drawdown metrics, completing the Observe stage.
2
Rule Injection & Risk Diagnosis Advisor Agent

Compare metrics against behavioral finance Skill definitions to diagnose biases, and trigger risk intervention on high-risk active positions.

File: trading_risk_coach/agents/advisor_agent.py
Logic: Reads SKILL.md rules; if the Disposition Effect Ratio (avg_loss / avg_win) is >= 1.5, it triggers a win-small-lose-big alert. If active positions lack stop losses, it executes risk mitigation tools (Stop-loss set / Emergency close), completing the Think & Act stage.
3
Zero-Trust Hard Guardrail Interception Guardrail Callback

Deterministic Python guardrail interceptor filters out high-risk trading advice before delivering reports to clients.

File: trading_risk_coach/guardrails/safety_rules.py
Logic: Text is passed through a registered after_model_callback. If the text contains gambling patterns ("averaging down", "holding losses"), it is intercepted and replaced with a fallback warning, ensuring zero-trust physical protection.
4
Formatting & Audit Opinion Signing Critic Agent

Review generated advice, ensure exact quantitative figures are cited, classify the account risk under Three-Value logic, and sign the official Quantitative Risk Audit Report.

File: trading_risk_coach/agents/critic_agent.py
Logic: Conducts a QA review to verify data points against source facts. It categorizes the risk status as 🟢 [GREEN - SAFE], 🟡 [YELLOW - WATCH], or 🔴 [RED - BREAKER], and formats the final quantitative opinion.

Safety & Operational Limits

# Demo Scenarios & Roadmap

Sample Query:

Please review my recent XAUUSD trades and tell me whether I am showing win-small-lose-big behavior.

Expected Agent Execution Flow:

MCP queries trade records and M1 candles -> Analysis Agent computes win rate / reward-risk / Disposition Ratio -> Advisor Agent queries SKILL.md -> Advisor triggers simulate_historical_risk_replay -> Guardrail sanitizes text -> Critic Agent signs final risk audit report.

Limitations & Future Roadmap:

Broker layers are simulated, relying on anonymized MT5 logs and XAUUSD M1 CSVs. Future iterations will integrate real-time broker APIs, database MCP servers, multi-symbol portfolio risk aggregates, and enterprise compliance dashboards.
