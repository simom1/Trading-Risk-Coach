本系统重点解决了以下三个散户交易员的致命痛点：

  • 无保护性止损硬扛：避免因为一笔持仓未挂止损遭遇行情突变而爆仓。
  • AI/人性的赌徒谬误：彻底在安全机制上切断 LLM 可能给出的“扛单”、“加仓摊平”等高危赌徒话术。
  • 处置效应（赢小亏大）：通过对历史成交数据的精准画像分析，警示持仓时长失衡与高频过度交易。

1 解决“无止损死扛”导致的爆仓风险 真实案例
真实情况 (Before)：硬扛两天，损失重创账户
交易行为：在 2026-02-21 01:37 开仓一笔大手数 XAUUSD（1.14手），开仓时未设置保护性止损。随后黄金行情一路单边逆行，交易员由于心理抗拒止损选择死扛。拖延 2.5 天（超3600分钟）后，在 2026-02-23 16:05 因保证金不足风险被迫手动割肉平仓，造成单笔惊人的 -$14,935.14 损失。

2 阻断 AI 建议中的“加仓摊平 / 扛单”赌徒谬误 安全防护
没有安全护栏时 (Before)：大模型受诱导给出有害的“加仓摊平”建议
安全隐患：当账户产生浮亏时，大模型（LLM）容易迎合交易员急于回本的焦虑心理，顺从逻辑给出“在更低位加仓拉低均价”或“继续硬扛等待反弹”等建议。这种建议实际上是在扩大仓位敞口，极易导致爆仓。
// AI 原始未拦截的危险回答： "分析到您当前的黄金多单出现浮亏，为了尽快挽回本金，建议您在金价下跌时加仓摊平成本。这样能把您的开仓均价从 5030.0 拉低到 5010.0。只要金价稍有反弹，您就能立刻回本出局。在此期间，请继续扛单，不要盲目止损..."
有安全护栏时 (After)：确定性 Python 安全防线拦截危险内容
系统控制：系统引入了 `safety_rules.py` 的物理安全拦截。即使大模型在推理层产生了加仓或死扛的冲动，其最终生成的建议文本也会在 **<1ms 内被正则硬过滤**，用强硬的警示和符合真实 2% 风险控制的科学规范进行替换，杜绝了有害信息到达用户端。
// 安全拦截器拦截后的合规回答： "[安全护栏已拦截原始建议] 检测到该建议可能包含高风险的仓位管理逻辑（例如加仓摊平、扛单等）。系统不会输出此类建议。 科学风控建议： 1. 严格遵守单笔交易风险不超过账户总资产 2.0% 的上限规则。 2. 请复盘您设置的原始止损线是否被破坏，如破坏必须立刻执行出场，切勿通过加仓稀释成本。"
⚠️ 风险成效：防范了人工智能生成危险建议导致交易员情绪化执行而扩大的次生灾害。
3 消除“拿了芝麻，丢了西瓜”的处置效应行为 去偏分析
无系统复盘时 (Before)：交易员陷入“赢小亏大”的心理泥潭
病态模式：真实数据中，交易员有很多笔持仓仅 1分钟左右、只赚了 $0.33 或 $0.40 就匆忙平仓的盈利记录（如 2026-05-12 11:39 的交易）。因为害怕盈利回吐，选择早早落袋为安。但面对亏损单，亏损单平均持仓却高达 **45.7分钟**。这导致了胜率很高，但平均亏损（-$443）几乎冲刷掉了所有平均盈利（+$447），账户成长陷入停滞。

引入系统复盘后 (After)：精准计算偏差系数，纠正快速止盈偏见
认知纠偏：Analysis Agent 精准分析 3,225 笔 XAUUSD 交易后，计算出当前账户的 **处置效应系数（Disposition Ratio）为 0.99**。Critic Agent 随即发出明确警示：胜率 58.6% 虽然不错，但是“快速止盈（均持仓仅1min）”极大限制了期望利润的发展。风控建议书强制纠错，指导交易员改变持仓风险期望结构。

系统如何运作？(Google ADK 多智能体闭环)
01
实时观测
Analysis Agent 通过 MCP 接口读取账户最新订单及波动率 (XAUUSD_M1.csv)。
02
诊断评级
Advisor Agent 依据 SKILL.md 设定的处置效应阈值进行评估，划定绿/黄/红三态状态。
03
主动执行
若订单缺止损，Advisor 自动触发 `execute_risk_mitigation` 强制修改或紧急强平订单。
04
确定性过滤
经过 Python 安全过滤硬阻断赌徒式分析，最终由 Critic 审计输出量化建议书。
