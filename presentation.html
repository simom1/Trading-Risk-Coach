Target Domain

Three Fatal Retail Trading Pitfalls

  • 1

    The Disposition Effect

    Traders cut profits quickly ("win small") to avoid loss anxiety, but hold losing positions for hours or days ("dead hold"), resulting in negative expected returns.

  • 2

    Missing Stop-Loss Protection

    Entering positions without protective boundaries. A single sharp market spike can destroy months of accumulated account profits.

  • 3

    Gambler's Fallacy & AI Hallucinations

    Under distress, traders average down or multiply lot sizes (Martingale logic). Unconstrained LLMs frequently hallucinate and support these emotional, toxic moves.

Agentic Pipeline

Google ADK Workflow Design

A sequential workflow where specialized agents query MCP tools, evaluate risk thresholds, and format safe outputs:

Analysis Agent

Calculates win rate, average win/loss, and Disposition Ratio

➔
Advisor Agent

Injects SKILL.md and triggers active mitigation tools

➔
Guardrail Filter

Python callback intercepts averaging down advice

➔
Critic Agent

Audits metrics and signs Three-Value report

Key Innovation: Decoupling the pipeline allows the Analysis Agent to focus strictly on numerical calculations while the Critic Agent acts as an independent compliance reviewer.

Dual-Directional Tools

FastMCP Server Capabilities

The agents communicate with the broker data layer using standard Model Context Protocol (stdio):

1. Read Tools (Trade Records & Market Candles)
• get_account_stats(days): Computes ratios from real_trades.csv. • get_market_context(): Queries real XAUUSD M1 price candles around timestamps. • simulate_historical_risk_replay(): Simulates historical risk paths.
2. Write Tools (Active Execution on Mock Broker)
• execute_risk_mitigation(action, ticket_id, param): - set_hard_sl: Place protective SL. - emergency_close: Immediate market exit.

All database exports (`real_trades.csv` with 3,225 records) and market K-lines (`XAUUSD_M1.csv` with 173K rows) represent real trading history.

Zero-Trust Output Layer

Preventing AI Hallucinations

Even if an LLM is induced to provide dangerous trading recovery advice, a deterministic Python interceptor callback filters the text in <1ms before delivery:

Raw LLM Response (Danger)
"You are currently in a draw-down. We suggest you average down your cost by adding buy positions as the price falls. This will lower your entry average price..."
Sanitized Output (Safe)
"[Security Guardrail Intercepted Original Suggestion] Detected high-risk position management logic. Falling back to safe template. Rules: 1. Keep risk exposure under 2.0%."

This ensures 100% compliance with risk management guidelines defined in SKILL.md.
